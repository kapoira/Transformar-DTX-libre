WAV to XA v1.20 

■このツールは
　WAVデータをBandJAMで使用可能なXAフォーマットで圧縮するツールです。
　手抜きのため、DOS窓ツールです。

■つかいかた
　無圧縮のPCMデータ(普通のWAV)のみ使用できます。
　コマンドラインで
　xa.exe ファイル名.wav
　これだけです。ファイル名の拡張子を.xaにしたファイル名でxaファイルを
　作成します。
　なお、ファイル名にはワイルドカードが使用できます。

■応用
　xa.exe /?
　でコマンドラインオプションの解説が表示されます。

　WAV to XA v1.20 Copyright 2000 LinnerSystem.
　Usage  : xa.exe <option> [filename<.WAV/.XA>]
　Option : -e[n]    Encode[WAV->XA](Default) / n:BitCount(4/6/8)
　         -d       Decode[XA->WAV]
　         -p       Play File
　         -o[dir]  Output Directory
　         -u       Update

　★-e[n]
　　指定したwavを圧縮します。デフォルトの動作です。
　　-eの後に続けて数字を指定すると圧縮ビット数を指定できます。
　　-e4 4bitで圧縮(ファイルサイズ小さい、音質悪い)
　　-e6 6bitで圧縮(ファイルサイズ普通、音質普通)
　　-e8 8bitで圧縮(ファイルサイズ大きい、音質良い)
　　数字を省略した場合は6bitがデフォルトになります。
　　出力ファイルの拡張子は.XAになります。

　★-d
　　圧縮されたXAをWAVに戻すときに使用します。
　　出力ファイルの拡張子は.WAVになります。

　★-p
　　指定したXAデータを再生します。
　　再生中になにかキーを押すと中断します。

　　/e /d /p は同時に指定できません。
　　同時に指定した場合、コマンドラインの一番後ろにある指定が採用されます。


　★-o[ディレクトリ名]
　　XA/WAVの出力先のディレクトリを指定します。
　　ディレクトリは用意しておいてください。自動作成はされません。
　　これを指定しない場合、ソースのディレクトリと同じディレクトリに
　　出力されます。

　★-u
　　出力ファイルが既に存在する場合、ソースとタイムスタンプを比較して
　　変更のあったもののみを処理します。

■例
　xa.exe -oxa -e8 *.*
　カレントディレクトリのwavデータ全てを8bitで圧縮して、xaディレクトリに
　出力します。
　こんな感じで。
　windowsからパソコン始めた人にはツライかもしれませんね(^^;

■おまけ 圧縮のコツ
　4bit圧縮だと、ファイルサイズが1/4強になりますが、高音域の音にはプチプチと
　ノイズが入ってしまうようです。
　その場合は、6bit/8bitで圧縮してみてください。

　また、8bitWAVデータは内部で一度16bitに展開した後、圧縮をかけるので、
　ソースとなるWAVデータは16bitで用意することをオススメします。

■おまけ MP3のこと
　このゲーム(BandJAM)では、圧縮されているwavは、codecを使用してデータを
　展開するようにしています。
　このため、mp3も展開できるわけですが、私的にはオススメしません。
　というのは、wavをmp3でencode->decodeしてみると、データの先頭に
　ほんのちょっと無音な時間が挿入されてしまいます。
　どのデコーダ／エンコーダの組み合わせでもこの現象はおきるのでmp3の
　仕様なんでしょうが、こうなるとギターなどで音が途切れるなど、非常に聞くに
　耐えないものになってしまいます。
　BM98あたりだと、前の音が消えるという概念がないのでそれでも大丈夫
　なんですが(それでも発音のタイミングはちょっとズレてるはずなんですが…)

　実は、開発中バージョンでは、MP3展開ルーチンを自前で持ってたりしましたが、
　上記のような現象が起きて使い物にならないので泣く泣く削除しました…。
